# S3 Media Store

External media on AWS S3. Best for sites already using AWS infrastructure.

## Install

```bash
pnpm add next-tinacms-s3 @tinacms/auth @aws-sdk/client-s3
```

## Credentials

```env
# .env
AWS_ACCESS_KEY_ID=your-access-key
AWS_SECRET_ACCESS_KEY=your-secret-key
AWS_REGION=us-east-1
S3_BUCKET=your-bucket-name
```

Use IAM credentials with PutObject/GetObject permissions on the specific bucket.

## `tina/config.ts`

```typescript
export default defineConfig({
  // ...
  media: {
    loadCustomStore: async () => {
      const pack = await import('next-tinacms-s3')
      return pack.TinaCloudS3MediaStore
    },
  },
  // ...
})
```

## API route

```typescript
// app/api/s3/[...path]/route.ts
import { createMediaHandler } from 'next-tinacms-s3/dist/handlers'
import { isAuthorized } from '@tinacms/auth'

const handler = createMediaHandler({
  config: {
    region: process.env.AWS_REGION!,
    credentials: {
      accessKeyId: process.env.AWS_ACCESS_KEY_ID!,
      secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY!,
    },
  },
  bucket: process.env.S3_BUCKET!,
  authorized: async (req: any) => {
    if (process.env.NODE_ENV === 'development') return true
    try {
      const user = await isAuthorized(req)
      return user && user.verified
    } catch {
      return false
    }
  },
})

export { handler as GET, handler as POST, handler as DELETE }
```

## Public URL pattern

S3 URLs follow:

```
https://<bucket>.s3.<region>.amazonaws.com/<key>
# OR with CloudFront:
https://<your-cloudfront-id>.cloudfront.net/<key>
```

For production, put CloudFront in front for CDN + caching.

## next/image config

```typescript
images: {
  remotePatterns: [
    { protocol: 'https', hostname: '*.s3.*.amazonaws.com' },
    { protocol: 'https', hostname: '*.cloudfront.net' },
  ],
}
```

## Bucket configuration

Bucket should:

- Allow public read (for serving images)
- Block public list (no directory browsing)
- CORS configured for your domain (if uploading from browser)

CORS:

```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["GET", "PUT", "POST"],
    "AllowedOrigins": ["https://your-site.com", "http://localhost:3000"],
    "ExposeHeaders": []
  }
]
```

## DigitalOcean Spaces (S3-compatible)

DO Spaces works with the same `next-tinacms-s3` package — just point at DO endpoints. See `references/media/05-do-spaces.md`.

## Cost

S3 charges:

- Storage: ~$0.023/GB/month
- GET requests: $0.0004/1000
- Egress: $0.09/GB (more if not behind CloudFront)

For a typical marketing site: < $5/month. Cheaper than Cloudinary's paid tier for high-volume use.

## Common mistakes

| Mistake | Effect | Fix |
|---|---|---|
| IAM with too-broad permissions | Security risk | Scope to specific bucket + actions |
| Bucket without CORS | Browser uploads fail | Add CORS config |
| Missing `remotePatterns` for next/image | Images don't render | Add S3/CloudFront hostnames |
| Used signed URLs without expiry | Tokens leak | Don't include in committed content |
| Egress costs high | No CloudFront | Add CloudFront in front |
