# Manufact Cloud (mcp-use Cloud)

Fastest path to production — `mcp-use deploy` builds and deploys from your GitHub repo.

---

## 1. Setup

```bash
# Install the CLI (skip if you used create-mcp-use-app)
npm install -g @mcp-use/cli

# Login to Manufact Cloud
npx mcp-use login

# Deploy
npx mcp-use deploy --name my-server
```

Add a deploy script to `package.json` if you prefer:

```json
{
  "scripts": {
    "dev": "mcp-use dev",
    "build": "mcp-use build",
    "start": "mcp-use start",
    "deploy": "mcp-use deploy"
  }
}
```

## 2. GitHub-detected deploy

If your project is a git repo with a remote, `mcp-use deploy` detects it:

```
GitHub repository detected:
  Repository: your-org/your-repo
  Branch:     main
  Commit:     abc1234

Deploy from GitHub repository your-org/your-repo? (y/n):
```

The build runs from the **remote HEAD on GitHub**, not your working tree. Uncommitted or unpushed changes are invisible to the deploy.

```bash
git add -A && git commit -m "deploy: latest" && git push
npx mcp-use deploy
```

If you skip the push, the deploy succeeds but the production server runs the previous commit. See `27-troubleshooting/01-error-catalog.md` row "Deploy missing recent changes".

## 3. `.mcp-use/project.json` — track this in git

`.mcp-use/project.json` links your local project to a specific cloud deployment. Without it, every deploy from a different machine assigns a **new subdomain** and breaks any custom domain (CNAME) you configured.

In `.gitignore`:

```
.mcp-use/
!.mcp-use/project.json
```

Commit it once after the first deploy:

```bash
git add -f .mcp-use/project.json
git commit -m "track deployment link"
git push
```

Do NOT pre-create `project.json` manually. The first deploy writes it.

If the cloud-side server was deleted (FK constraint error on deploy), delete `project.json` locally and redeploy to create a fresh link.

## 4. Deploy flags

```bash
npx mcp-use deploy \
  --name my-server \
  --org acme-corp \
  --env DATABASE_URL=postgres://... \
  --env-file .env.production \
  --open
```

| Flag                  | Notes                                                      |
|-----------------------|------------------------------------------------------------|
| `--name`              | Label only. URL subdomain is auto-generated by Manufact Cloud (e.g. `black-silence-ot8sz`). |
| `--org`               | Target org slug or id. See `25-deploy/05-cli-and-org-management.md`. |
| `--env`, `--env-file` | Sets env vars on the deployed server.                      |
| `--root-dir`          | Monorepo: path to the server package.                      |
| `--runtime`           | `node` (default) or `python`.                              |
| `-y` / `--yes`        | Non-interactive (CI).                                      |

## 5. After deploy

```
✓ Deployment successful!

🌐 MCP Server URL:
   https://your-subdomain.run.mcp-use.com/mcp

🔍 Inspector URL:
   https://inspector.manufact.com/inspector?autoConnect=https://your-subdomain.run.mcp-use.com/mcp
```

Verify:

1. `curl -s https://{url}/health | jq .status` returns `"ok"`.
2. Open the Inspector URL — confirm tool list and call one tool.
3. Update Claude Desktop / Codex / Claude Code configs:
   ```json
   {
     "mcpServers": {
       "my-server": {
         "url": "https://your-subdomain.run.mcp-use.com/mcp"
       }
     }
   }
   ```
4. If you have a CNAME, verify it still resolves: `curl -s https://your-domain.com/health`.

## 6. Test locally first

```bash
npm run build && npm run start
```

Open `http://localhost:3000/mcp` in the Inspector. If it works locally, the cloud build will work — Manufact Cloud uses the same `mcp-use build` and `mcp-use start` commands.

## 7. Common failures

| Symptom                                          | Fix                                                       |
|--------------------------------------------------|-----------------------------------------------------------|
| Deployed server missing recent changes           | `git push` before `mcp-use deploy`.                       |
| New subdomain on every deploy / custom domain breaks | Track `.mcp-use/project.json` in git.                  |
| FK constraint error on deploy                    | Cloud server deleted — delete local `project.json`, redeploy. |
| 401 on deploy                                    | API session expired — run `npx mcp-use login` again.     |
| `dist/mcp-use.json` missing                      | Run `npm run build` first.                                |

Full catalog: `27-troubleshooting/01-error-catalog.md`.
